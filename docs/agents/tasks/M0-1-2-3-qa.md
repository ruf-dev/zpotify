# Task Brief — M0-1/2/3 · QA (upload flow end-to-end)

**Area:** QA
**Priority:** high
**Goal:** Verify the complete upload flow — file → backend → DB → Create Song form — is correct, convention-compliant, and rejection-safe.

---

## What was shipped (diff scope)

- **Backend:** `POST /api/upload/audio` HTTP handler in `internal/transport/wapi/`, `UploadFile` service method, `InsertFileMeta` SQLC query
- **Frontend:** `UploadProcess`, upload widget wired to progress + dialog trigger, `CreateSongWidget`, `CreateSongDialog`, `SongProcess.createSong`, `ArtistProcess.listArtists`

---

## Test checklist

### Backend — happy path
- [ ] `POST /api/upload/audio` with valid MP3 + valid auth token → 201 + `{"file_id": N}`
- [ ] DB: `files_meta` row exists with correct `path`, `size`, `uploaded_by`, `verified=false`
- [ ] File exists on disk at the path stored in DB

### Backend — rejection cases
- [ ] No auth token → 401
- [ ] Auth token for user without `can_upload` → 403
- [ ] Unsupported MIME type (e.g. `image/jpeg`) → 400
- [ ] File exceeding max size → 400
- [ ] Malformed multipart body → 400

### Backend — convention scan (against git diff)
- [ ] No raw SQL strings in Go files — all queries via SQLC generated functions
- [ ] No hand-edited files under `internal/storage/pg/` or `internal/api/server/zpotify_api/`
- [ ] All `Close()` calls are `utils.CloseWithLog(...)`, never `defer c.Close()`
- [ ] No inline struct literals passed directly to function calls
- [ ] Errors at storage layer use `rerrors.Wrap(err)`; service layer uses `service_errors/` vars

### Frontend — happy path
- [ ] File picker opens on click
- [ ] Selecting a file starts upload and progress bar advances
- [ ] Progress reaches 100% and Create Song dialog opens automatically
- [ ] Title field is pre-filled with filename (extension stripped)
- [ ] Artist multi-select populates from backend; typing filters the list
- [ ] Submitting calls `SongAPI.CreateSong`; dialog closes on success

### Frontend — rejection / edge cases
- [ ] Upload failure shows error state (not a blank screen)
- [ ] Create Song submit with empty title is blocked (client-side validation)
- [ ] Network failure during artist fetch does not crash the form

### Frontend — convention scan (against git diff)
- [ ] No `fetch()` or gRPC calls inside any `widgets/`, `components/`, `pages/`, or `dialogs/` file — only in `processes/`
- [ ] No inline styles anywhere
- [ ] No hardcoded hex colour values or px values — only CSS variables
- [ ] All components are named functions (`function Name()`), not arrow-function `const`
- [ ] Each file contains exactly one exported component
- [ ] Dialogs opened only via `useDialog` — no conditional `{show && <Dialog />}` pattern
- [ ] All imports use `@/` alias

### Build gates
- [ ] `go build ./...` passes with zero errors
- [ ] `npm run build:ui` passes with zero TypeScript errors
